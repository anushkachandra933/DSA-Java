What are function?
-It is a block of reusable code that performs a specific task.


Difference between arguments and parameters
-Arguments are actual value passed to a function during function calling
whereas,Parameters are variables passed to a function during function definition


Function overloading and Function overriding
-Function overloading is a type of compile-time polymorphism in which multiple functions have same name but differ in number of parameters or type of parameters or order of parameters.
Function overriding is a type of runtime polymorphism in which we provide new implementation of a function in a subclass that already exist in its superclass.(Annotation - @Override)


How function uses stack and heap memory-
function calls and its local variables are stored in stack frame.
Actual object is stored in heap memory,static functions and variables are stored in a special area inside heap called method area or static area.

why do we use function?
-for reusability,modularity,avoiding repetition,easy to debug and test.


Types of function :- 
Parameterised function - A function that takes parameters.
Non-Parameterised function - A function that doesn't have parameters.


Recursive function - A function calling itself within its own body.


Context of function - Global, Lexical, Functional and Local scope


Components of function-Access modifier,return type,function name,parameters,function body


Sequential flow of function - code is executed line by line from top to bottom


Call by value - In this,we pass primitive data types as parameters to a function.Changes inside the function will not persist outside.
Call by reference - In this,we pass non-primitive data types as parameters to a function.
Changes inside the function will persist outside also.
